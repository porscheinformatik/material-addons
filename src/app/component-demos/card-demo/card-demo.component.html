Cards group related content and actions into a stable section, such as a read-only summary, editable form, or expandable detail area.

When To Use

  • Group fields or content that users understand as one section.
  • Switch a section between read-only and edit modes.
  • Hide secondary detail until users expand it.
  • Place contextual actions close to the content they affect.

Behavior

  • Read-only cards can show content without edit or footer actions.
  • Editable cards show an edit action until editMode is enabled.
  • Edit mode shows save and cancel actions when readonly is false.
  • Expandable cards can collapse content unless edit mode is active.
  • An additional header action is shown when additionalActionIcon is provided.
  • The card can render without a title when the surrounding page already provides enough context.

Import the CardComponent and use it via <mad-card> .

Examples

Design Notes

  • Use cards to create meaningful content groups, not as decoration around every element.
  • Keep header actions limited to actions that affect the whole card.
  • Use expandable cards for secondary details, not for required information.
  • Check spacing and density when placing cards inside dashboards or narrow side panels.
