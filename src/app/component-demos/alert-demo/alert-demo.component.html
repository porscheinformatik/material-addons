Alerts communicate system feedback, status changes, and important contextual messages without forcing users out of their current workflow.

When To Use

  • Confirm that an action succeeded or failed.
  • Explain a warning before the user continues a workflow.
  • Show contextual information that should stay close to the related content.
  • Offer a short follow-up action such as reviewing details or undoing a change.

Behavior

  • Use type to communicate semantic intent: success, info, warning, or error.
  • Use size to tune density for the surrounding layout.
  • Provide actionText only when the message has one clear follow-up action.
  • Set [closeable]="false" when the message must remain visible.
  • Keep messages concise. Long messages wrap, but detailed explanations usually belong in page content.

Import the AlertComponent and use it via <mad-alert> .

Design Notes

  • Choose the alert type by meaning, not by preferred color.
  • Use warning and error alerts sparingly so they keep their visual weight.
  • Check the Colors page when validating alert contrast across available themes.
  • Prefer short action text so the alert remains readable in narrow layouts.
