Tiles present compact status, category, or metadata labels with consistent color, sizing, and optional icon support.

When To Use

  • Show short semantic labels in dashboards, summaries, or dense business views.
  • Make status values easier to scan without adding a full alert message.
  • Pair a short label with an icon when the icon improves recognition.

Behavior

  • Use variant for semantic meaning: success, info, warning, or error.
  • Use size to match the surrounding density.
  • Use icon only when it reinforces the label.
  • Use iconPosition to place the icon before or after the label.
  • Keep labels short. The tile component is designed as a compact inline indicator.

Import the TileComponent and use it via <mad-tile> .

Examples

Design Notes

  • Use tile colors consistently with status meaning across the application.
  • Do not use tiles for long explanatory messages; use page content or alerts instead.
  • Check the Colors page when reviewing theme-specific tile colors.
  • Use icons sparingly in dense lists so they remain helpful rather than decorative.
