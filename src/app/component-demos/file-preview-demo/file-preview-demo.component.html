The File Preview Component allows you to display uploaded files as thumbnails with optional overlay preview, download, and delete actions.


Supported File Types

  • Images — JPG, JPEG, PNG, GIF, WEBP, BMP, SVG, ICO (thumbnail generated, full overlay preview) — No additional dependencies required
  • PDF — inline overlay preview (native browser rendering). Rasterised thumbnails require pdfjs-dist (optional)
  • DOCX — thumbnail and inline overlay preview. Requires docx-preview (optional peer dependency)
  • Excel / CSV / ODS — thumbnail and inline overlay preview with search, filter, and sort. Requires @e965/xlsx (optional peer dependency)
  • Other types — displayed with a generic file icon; no preview available

Optional Dependencies

Some preview features require optional peer dependencies. Install only what you need:

  • npm install pdfjs-dist — for rasterised PDF thumbnails (not required for inline PDF preview)
  • npm install docx-preview — for DOCX preview and thumbnails
  • npm install @e965/xlsx — for Excel/CSV/ODS preview and thumbnails

If an optional dependency is not installed, the component gracefully falls back to showing a file icon or error message instead of crashing.


Behavior

  • Renders image thumbnails automatically from a URL, Blob, File, ArrayBuffer, or Base64 input
  • Opens a responsive overlay dialog for images, PDFs, DOCX and Excel workbooks when the thumbnail is clicked (inline rendering in supported formats)
  • Falls back to a file-type icon when a preview is unavailable (unsupported format or missing optional dependency)
  • Action bar below each thumbnail shows configurable preview, download, and delete buttons
  • Supports live updates through Angular's change detection when the items input is updated
  • Gracefully handles missing optional dependencies with error messages in the preview overlay

Import the FilePreviewComponent in your component and use it via <mad-file-preview> .
