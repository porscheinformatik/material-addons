Breadcrumbs show where the current page sits in a hierarchy and give users a lightweight way to move back to parent levels.

When To Use

  • Show location inside a deep application hierarchy.
  • Help users move from a detail page back to a parent list or section.
  • Expose the current context when page titles alone are not enough.
  • Provide a copy action for workflows where users share the current context.

Behavior

  • Items with route use Angular router navigation.
  • Items with href render as external links.
  • Items without route or href render as current-page text.
  • The copy action is hidden by default and shown when [showCopy]="true".
  • Keep labels short enough to scan as a path.

Import the BreadcrumbComponent and use it via <mad-breadcrumb> .

Examples

Design Notes

  • Use breadcrumbs as secondary navigation, not as the main page navigation.
  • Use the current item to reinforce page context without making it look clickable.
  • Limit hierarchy depth so the breadcrumb remains readable on narrow screens.
  • The separator and current item color can be themed with the breadcrumb CSS custom property.
