Use to organize large datasets that users can scan, compare, filter, sort, paginate, and act on.

Use When

  • Displaying structured row-and-column data
  • Supporting filtering, sorting, pagination, row actions, configurable columns, or custom row templates
  • Handling local datasets or externally paged server-side data

Behavior

  • Supports synchronous and asynchronous data loading
  • Supports table-level and column-level filtering
  • Supports pagination, sorting, row actions, batch selection, child rows, expandable rows, and column configuration
  • Supports custom cell templates through madColumnDef and expandable templates through madExpandableColumnDef

Import the DataTableComponent and use it via <mad-data-table> .

Examples

  • Basic Table: Basic table with pagination and filter
  • Data transformation: Custom transformation of column data
  • Single Mode: Table with single mode actions
  • Batch Mode: Table with batch mode actions
  • Async Loading: Table with asynchronous loading where sorting, filtering, and paging are handled server-side
  • Row ID: Return IDs instead of full rows on events
  • Child rows: Use child rows with limited support
  • Column configuration: Allow users to view and edit different column configurations
  • Table height: Stretch the table to its parent height
  • Column filter: Filter table data from filterable column headers
  • Stateful datatable: Persist sort, filter, and page-size state
  • Cell templates: Render custom cell content
  • Expandable rows: Render expandable row detail content

Basic table

The data table expects the following input:

  • displayedColumns: definitions of columns of type DataTableColumn with the following properties
    • label (required): label displayed in the table and translated when possible
    • dataPropertyName (required): property name of the column in the row data
    • isSortable (optional): enables sorting for the column
    • isRightAligned (optional): aligns column text to the right
    • transformer (optional): transforms data before display. See Data Transformation.
    • transformerParams (optional): params for the transformer function

Data transformation

To transform displayed data, set the optional transformer and transformerParams properties in the column definition. You can supply any function as transformer.

  • Example: custom transformations for Title translation, Gender symbol display, and Email uppercase display
  • Internal sort and filter use the displayed data, not the original data. For example, after translating titles, the filter will match translated values instead of the original values.
  • events, like row actions, still return the original row data

Single mode

If at least one row action exists with type SINGLE and no batch action exists, the table is displayed in single mode.

  • All row actions are available in the action menu of the leftmost column
  • Clicking any row triggers the default action, which is the first action
  • Actions with type NONE are independent of rows and are displayed above the table, for example a create action
Back to overview

Batch mode (multiselect)

If at least one row action exists with type BATCH, the table is displayed in batch mode. You can force the display mode; the example below uses the batch selection button to switch between batch and single modes.

  • Clicking a row selects or deselects the row
  • All row actions are displayed above the table
  • Action buttons are disabled when the amount of selected rows does not match the action type
  • Batch actions show an indicator with the number of selected rows
  • Clicking the checkbox in the header selects all rows of the current page

Asynchronous loading

Use asynchronous loading for large datasets when only one page of data should be loaded at a time.

  • Because the table only has one page of data, filtering, sorting, and paging must be handled externally
  • filtering: the internal filter is deactivated; create your own filter and update [page] and [tableData]
  • sorting: subscribe to (sortEvent), load the new page from the server, then update [page] and [tableData]
  • paging: set the current page with [page] and subscribe to (pageEvent), then load the requested page and update [page] and [tableData]
  • Set [loading] to true while loading data to display a spinner

IDs instead of rows

By default, row actions return the complete selected rows in the event. If events should return IDs instead of full row objects, provide an ID generator function to [idGenerator].

Back to overview

Child rows (limited support)

Rows can be marked as child rows. Child rows currently have limited support:

  • Table actions for child rows are ignored in batch and single actions
  • Automatic sorting does not work reliably for child rows; implement sorting logic externally when needed
  • Provide [idGenerator] and [parentIdGenerator]. Rows with a generated parent ID are treated as child rows.
Back to overview

Custom column configuration

If you provide at least one definition for [columnDefinitions], the table displays a button for users to configure columns. The definition can have a name, defaults to USER if omitted, and must define all available and selected columns.

The data table emits (columnDefinitionChangeEvent) when a column definition is saved or deleted. Subscribe to this event when the user's column configuration should be persisted.

If you provide more than one column definition template, users can choose which template they want to edit.

Stretch table to parent height

Set [tableClass] to datatable-vertical-stretch-to-parent to stretch the table to its parent height. The table has a minimum height and shows a scrollbar if it cannot display all rows while keeping pagination visible.

Back to overview

Filter table based on columns

Use [filterMode] set to COLUMN_BASED to filter a table from its columns. Add isFilterable to each filterable column. The filterParams.filterOptions property can pass key-value options for predefined filters.

Back to overview

Stateful datatable

Set [persistenceConfig] with persistSort, persistFilter, and persistPageSize to persist the table's sort, filter, and page-size state. Override DataTablePersistenceService and provide it through DATA_TABLE_PERSISTENCE_SERVICE_PROVIDER to use a custom implementation.

Back to overview

Cell templates for custom cell definitions

Use madColumnDef to define a column definition using the id of the DataTableColumn. Use madCellDef to define the matching cell definition. Use the let element syntax inside madCellDef to access the current row data.

Back to overview

Template to make a row expandable

Use madExpandableColumnDef to define an expandable definition. Use madExpandableCellDef to define the matching expandable cell definition. Use the let element syntax inside madExpandableCellDef to access the current row data. Use [rowExpandable] to define which rows are expandable.
