Use buttons to trigger actions with consistent Material Addons styling. The wrapper components keep the existing mad-* selectors, while the internals use Angular Material 21 button appearances.

Use When

  • Creating a primary page action such as save or continue
  • Providing secondary actions such as cancel, close, or back
  • Showing destructive actions that need a danger treatment
  • Rendering tertiary text actions or icon-only actions
  • Grouping related actions into a compact action bar

Behavior

  • Wrapper components render native buttons with Angular Material 21 appearances such as matButton="filled".
  • mad-outline-button supports primary, accent, and warn color variants.
  • madButton applies Material Addons outline, color, and uppercase styling to Angular Material buttons.
  • Disabled buttons use the native disabled state and keep their branded color identity through disabled theme tokens.

Import the ButtonModule in your app.module.ts and use the existing selectors such as <mad-primary-button> or the madButton directive on Angular Material buttons.
