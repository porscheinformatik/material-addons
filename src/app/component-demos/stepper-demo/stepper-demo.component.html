A stepper allows to create workflows that divides content into logical steps

This stepper component is based on the CdkStepper implementation.
Please find the documentation here: CdkStepper


Use When

  • work processes may not fit in one viewport
  • creation of workflows and separation of content into logical steps is needed
  • there is a linear workflow

Behavior

  • initially the first step is open
  • steps can be accessed either through the navigation button inside the step or by clicking on the step header
  • moving to the next step is only possible when the current step is completed/valid
  • steps are highlighted if they are valid/completed or invalid

Linear stepper

The stepper only allows a linear processing of the steps. The user has to complete the previous steps before proceeding. For each step, the stepControl attribute has to be set which is used to check the validity of the step.


Navigation

The stepper implementation already takes care of the navigation of the stepper. A step can either be accessed by the button inside the step or by clicking on the header. Alternatively, there is the possibility to define the navigation by using onNext/onDone/onHeaderClick. Then the navigation has to be triggered manually. You can use the @ViewChild() annotation to get a reference of your StepperComponent, which allows you to call the following method:

  • next() - navigate to the next step

Import the StepperModule in your app.module.ts and use it via <mad-stepper> .
